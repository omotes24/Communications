# JobTrack × Yell 面接モジュール準備仕様

現時点ではYell単独サービスとして動作し、JobTrackへの書き込み・JobTrack非公開企業CSVの複製・アカウント連携は行いません。将来、JobTrackがYellの面接機能を内包できるよう境界だけを定義しています。

## 想定する将来導線

```text
JobTrack企業詳細 → 面接を開始 → Yell面接モジュール
  ├─ 本人専用: 音声・全文文字起こし・面接履歴（将来のJobTrack Pro）
  └─ 管理者限定: 同意済み匿名体験談・過去問データ
```

企業はJobTrackの公開IDではなく、内部カタログ参照と同じ形式の `catalogRef`（`JT` + 6桁）で受け渡します。この値を公開URLやユーザー向け識別子として扱わず、Yell側へ企業マスタ本体をコピーしません。

## アカウント方式

起動契約は次の3方式を表現できますが、現在は `standalone` だけを使います。

- `standalone`: YellのSupabase Authで完結
- `shared`: 将来JobTrackと同一認証主体を使う場合
- `linked`: 別アカウントのまま、サーバー間で不透明な参照だけを紐付ける場合

紐付けテーブルは準備済みですが、ログイン連携API、UI、トークン交換、SSOは未実装・未有効です。方式が決まるまで実処理を追加しません。

## 面接体験談・過去問

文字起こしから、面接官が実際に質問したと判断できる項目だけを抽出します。氏名、学校、連絡先、面接官名、URL、候補者回答、企業機密は除外します。候補者が確認し研究利用へ同意したレポートだけを分析対象にします。

この機能は現時点で通常UIに存在しません。APIは非オーナーを管理者境界で拒否して
`XYZキーが必要です。` の403を返し、オーナーに対しても
`INTERVIEW_EXPERIENCE_ENABLED=false` の間は404を返します。

## 将来のJobTrack Pro面接アーカイブ

`interview_archives` はprivate Storageのobject path、保存同意、保持期限、削除状態を持つメタデータ契約だけです。音声・全文文字起こしのアップロード、再生、表示、Pro権限判定はまだ実装しません。

実装時の必須条件:

- 面接開始前に録音・保存の明示同意を取得する
- 保存しない利用を常に選択可能にする
- private bucketと短時間の署名URLを使う
- JobTrack Pro権限はサーバーで毎回検証する
- 本人による個別削除・一括削除・保持期限を提供する
- 匿名体験談への転用は別同意にし、アーカイブ同意と束ねない
