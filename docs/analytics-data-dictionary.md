# 分析データ・CSV仕様

管理者限定の `/api/admin/export/{dataset}?days=90` から、UTF-8 BOM付きRFC 4180形式で出力します。期間は1〜365日、最大50,000行です。上限到達時はレスポンスヘッダー `X-Export-Truncated: true` を返します。

## データセット

| dataset                 | 粒度                    | 主な用途                                       |
| ----------------------- | ----------------------- | ---------------------------------------------- |
| `customers`             | 匿名ユーザー×指定期間   | LTV、購入回数、残高、消費量、原価              |
| `purchases`             | 購入1件                 | 売上、プラン、購入トークン、リピート分析       |
| `usage`                 | AIリクエスト1件         | 機能・モデル・トークン・音声秒・検索・推定原価 |
| `daily`                 | 日×機能×モデル          | 時系列予測、異常検知、需要・原価予測           |
| `events`                | プロダクトイベント1件   | 導線、継続、機能利用、実験分析                 |
| `interview_experiences` | 同意済み体験談の質問1件 | 企業・選考段階別の質問傾向、難易度、準備項目   |

全CSVでユーザーは `yfy_` から始まるHMAC/SHA-256匿名IDです。安定した結合キーが必要な本番環境では `ADMIN_AUDIT_HMAC_SECRET` を必ず設定します。

## 機械学習での扱い

- ラベル漏洩を避けるため、未来の購入・退会情報を過去時点の特徴量へ混ぜないでください。
- `daily` のゼロ件の日も学習側で補完し、タイムゾーンはUTC保存・分析時にJSTへ変換します。
- `estimated_openai_cost_usd` は価格表と記録使用量による推定値で、請求書の正解ラベルではありません。
- `reasoning_tokens_subset` は `output_tokens` に内包される部分集合です。合計時に足し直さないでください。
- 面接体験データは投稿者同意済みかつ匿名化済みだけを出力します。質問数の少ない企業・職種は再識別リスクがあるため、公開集計では最小件数しきい値を設けてください。
- 生の応募結果、属性、学校名などを将来追加する場合は、目的・同意・保持期限・公平性評価を先に定義します。

## 収集しない情報

分析イベントにはIPアドレス、User-Agent、参照元URL、クエリ文字列、メール、氏名、プロンプト本文、生音声、全文文字起こしを保存しません。面接体験談CSVにも候補者の回答本文と文字起こしハッシュを含めません。

`analytics_events.metadata` は固定許可リストだけを受け付け、任意JSONをブラウザから保存できない設計です。行動分析は `NEXT_PUBLIC_PRODUCT_ANALYTICS_ENABLED=true` のときだけ送信します。

## Webアクセス分析

管理者限定の `/admin/analytics` は `web_analytics_page_views` をDB側RPCで集計し、7日・30日・90日のページビュー、匿名セッション、日別推移、ページ別・デバイス別の内訳を表示します。管理APIは集計値だけを返し、生のセッションハッシュはブラウザへ返しません。

- 未ログイン訪問も計測対象です。
- タブ単位で `sessionStorage` に生成したランダムIDを、サーバー側でHMAC/SHA-256化して保存します。
- `/admin` 以下は送信側と集計側の両方で除外します。
- 生のページビューは90日で削除します。
- 公開プライバシー画面からブラウザ単位で無効化できます。
- Do Not Track / Global Privacy Controlが有効なブラウザからは送信しません。
